> **日期**：2026-02-20
> **视角**：本文档记录的是当前已知的探索方向，而非封闭的功能全集。系统将随 Maeiee 的认知持续生长。

## 缘起：一个极客的终生浪漫

在这个应用生生灭灭、云服务随时可能关闭、技术栈几年一换的时代，Maeiee 作为一个技术 Geek，一直在思考一个问题：**有没有一个生产力系统，能够伴随一个人的一生？**

他不需要一个迎合大众的"超级 App"，也不想被任何封闭的商业生态绑架。他渴望的是一个为自己量身定制、能够迭代扩展终生的数字底座。

这个系统应该能跨越 Maeiee 的不同人生阶段，跟随技术的演进而成长。三十年后，当当下最时髦的前端框架早已销声匿迹时，他的数据依然完好无损，他的肌肉记忆依然有效，他的系统底座依然在稳健运转。

为了实现这个愿景，Maeiee 为自己设计了这套系统。

## 核心哲学：变与不变

为了让系统存活数十年，Maeiee 在设计上将「永恒」与「短暂」进行了严格的剥离。

### 1. 数据的永恒：文件即数据库

借鉴 Obsidian 的哲学，所有核心数据必须得到最妥善的保存，不锁定于任何专有格式。

- **文件为绝对真相源**：本地文件（如纯文本、Markdown 等）是数据的最终归宿
- **分层存储**：文件以 UUID 命名，按前缀分层组织；Sidecar 文件存储结构化元信息
- **随用随建的索引**：每台设备本地维护基于 SQLite 的索引。索引随时可被销毁，并能从文件中完美重建。不依赖任何云端黑盒

**概念详解**：详见 [核心概念 - 数据层](./concepts.md#1-数据层)。

### 2. 能力的流转：Emacs 式统一注册

技术栈会不断更迭，但「能力」本身是中立的。系统就像是一个更现代化的 Emacs：

- **能力总线**：来自不同技术栈（Rust、Swift、C#、Python...）的底层能力，最终都被统一注册到一个总线上
- **代码生成 SDK**：无论未来出现什么新的编程语言，只需通过自动化代码生成对应的 SDK，即可无缝接入当前系统
- **Intent 协议**：标准化的「意图表达」机制，让「打开 PDF」、「翻译文本」等需求脱离了具体应用，变成了系统级的通用命令

### 3. 视图的融合：无视技术栈的 GUI 积木

UI 框架的生命周期往往是最短的。Maeiee 不希望自己的工作流被特定的 GUI 技术限制：

- **底层渲染视图池**：不再区分技术栈，而是建立一套更底层的技术协议
- **融合式布局**：由不同技术栈编写的视图组件，能够像积木一样在同一个界面下进行融合渲染，实现可组合的视图布局

## 目标用户

**唯一用户**：Maeiee。

这是一个为自己量身定制的系统，不考虑商业化、不考虑大众市场、不考虑「开箱即用」。Maeiee 既是最苛刻的产品经理，也是唯一的开发者，更是唯的一用户。

如果这套系统的设计理念恰好也能帮助到其他技术 Geek，那是意外的收获。但系统演进的唯一决策标准是：**是否符合 Maeiee 当下的需求。**

## 核心价值主张

| 价值 | 说明 |
|------|------|
| **数据自主** | 文件为真相源，Maeiee 完全控制数据，不依赖任何云服务 |
| **能力开放** | 任何能力都可以注册，任何能力都可以调用 |
| **积木组合** | 小应用可以像乐高一样互相拼接，产生 1+1>2 的效果 |
| **跨设备** | 同一套设计理念在 Windows、macOS、Linux 上运行 |
| **AI 友好** | 架构天然支持 AI Agent 参与操作和编排 |

## 与"超级 App"的区别

这不是一个产品，而是一种个人数字生活方式。

| 维度 | 商业超级 App | Maeiee 的终生系统 |
|------|---------|-----------|
| **存在目的** | 占领用户时间，构建商业护城河 | 沉淀个人数据，构建极客的数字大脑 |
| **功能边界** | 追求大而全，将所有功能塞入一个二进制 | 多个小应用各司其职，产生 1+1>2 的涌现效应 |
| **扩展方式** | 等待官方施舍新功能 | 随时通过多语言 SDK 自己编写、随时热插拔 |
| **底层依赖** | 强依赖云端服务与数据库 | 依赖本地文件、开放标准与几十年寿命的基建技术 |
| **GUI 架构** | 单一技术栈锁定 | 跨技术栈的视图融合与自由拼接 |

## 当前探索坐标系

这套系统是一个有机的生命体。因此，这里没有传统的「短期/中期/长期规划」，只有 Maeiee 当前认知下的**探索方向**。

**当前锚点与方向**：

- **打通数据基建**：验证本地文件与 SQLite 索引的无缝双向映射，实现完全离线、跨设备的极速同步
- **协议定义**：确立 Intent 协议的初版，验证其在 Rust/Python/Swift 等至少三种语言环境下的互操作性
- **视图实验**：探索跨技术栈 GUI 融合渲染的具体技术方案与边界限制
- **应用积木化**：开发首批极其克制、单一功能的小工具，验证乐高式组装的可行性

**未来的留白**：

随着 Maeiee 技术的成长和生活重心的变化，这个系统可能会接入硬件设备、可能会演化出全新的视图形态、可能会接入未来十年的新一代 AI 架构。

**一切皆未定，一切皆开放。这只是一个终生旅程的起点。**

## 设计哲学

> "简单的东西可以组合出复杂的行为，但复杂的东西很难拆解成简单的部分。"

Maeiee 选择从最基础的简单设施出发：

- **最简的数据模型**：文件 + 元信息
- **最简的通信协议**：Intent 意图路由
- **最简的应用边界**：Do one thing and do it well

系统天然支持 AI Agent 的深度参与。因为当一切都被拆解为标准化的意图（Intent）和纯文本文件时，AI 就不再是一个悬浮的「聊天框」，而是能直接操作总线、编排工具链的数字助手。
